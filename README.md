> **System Requirements:**
>
> - Node.js v22.0.0 or higher
> - System build tools (gcc, make, python3)
> - 2GB RAM minimum (4GB recommended for optimal performance)
> - Linux Ubuntu 24.04 LTS or Debian 12 (required for liora-lib)

> [!IMPORTANT]
> **Optional Enhancements: `audio-decode` and `link-preview-js`**
>
> These packages are **not required** for core functionality and are treated as **optional peer dependencies**.
>
> - **`audio-decode`**: Enables waveform rendering for voice notes (PTT). Without it, audio still works‚Äîonly waveform visualization is disabled.
> - **`link-preview-js`**: Generates rich previews for URLs sent by the bot. Without it, links are still delivered‚Äîjust without metadata preview cards.
>
> This modular approach ensures Liora remains lightweight and deployable in constrained environments.
>
> **Install these packages if you want enhanced UI/UX. Skip them if you prefer minimal setup.**

## üçß Liora ‚Äì Advanced WhatsApp Bot Framework

<div align="center">
  <img src="https://readme-typing-svg.herokuapp.com?size=28&duration=3500&color=0B3D91&center=true&vCenter=true&width=600&lines=üå∏+Liora+‚Äì+Advanced+WhatsApp+Bot" alt="Liora Bot animated title" />
</div>

**Liora** is an enterprise-grade WhatsApp automation framework for developers who demand precision, modularity, and runtime-level control.

**Not beginner-friendly. Not plug-and-play. Unapologetically low-level.**

Built for developers who understand event loops, buffer management, and asynchronous architectures.

---

## üß± Core Capabilities

- **üõ†Ô∏è Native Performance** ‚Äî C++ bindings for media processing and format conversion
- **üß© Modular Architecture** ‚Äî Hot-reloadable handlers and customizable plugins
- **üìà Zero Overhead** ‚Äî Pure buffer-based memory flow
- **üß† Developer-Centric** ‚Äî Runtime control and system-level optimization
- **üîÑ Hot Reload** ‚Äî Update modules without restarting the bot
- **üóÉÔ∏è State Management** ‚Äî SQLite-backed atomic operations

---

## üìê Technical Highlights

- **üîì Fully Open Source** ‚Äî No obfuscation or telemetry
- **üì¶ Pure ESM** ‚Äî Async-first architecture, zero CommonJS dependencies
- **üü¢ Node.js v22+ Optimized** ‚Äî Leverages latest V8 JIT optimizations
- **üíæ SQLite-Backed** ‚Äî Atomic transactions and fast queries
- **üß¨ Extensible** ‚Äî Isolated, composable modules
- **üîå Plugin System** ‚Äî Dynamic functionality loading
- **‚ö° Baileys Integration** ‚Äî Stable multi-device WhatsApp support
